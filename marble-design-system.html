Layout Rules

页面骨架保持一致，列表页和详情页都能无缝套进去。

Topbar
56px 深色顶栏，承载面包屑、全局搜索、组织切换、通知和用户入口。
Sidebar
304px 多层树形导航，当前模块蓝色，当前页面灰底高亮。
Content Area
24px 页面内边距，标题区先行，后面接 toolbar、card、table、form。

Shared Components

这几类组件已经在列表页和详情页里统一了。

Page Header
标题、副标题、状态 badge、主操作按钮。
Toolbar Card
搜索、筛选、导出、分页、密度控制。
Content Card
所有业务区块统一白底卡片、细边框、轻阴影。
Data Table
sticky 表头、hover 行、主次信息双层排布。
Form Controls
12px 圆角、统一 focus ring、44px 输入高度。
Tabs
浅底容器 + 当前态浅蓝背景。
Stats Cards
详情页上方摘要指标，先展示关键业务信息。
Empty State
虚线框 + 人话说明，避免大片空白。

Section Header Spec

系统页、管理页、数据卡片统一用这套 section header，不再每块各写一套标题风格。

  • 字距统一 0.05em，作为 section label，不走页面主标题风格。
  • 头部高度统一，推荐 min-height: 52px。

Usage Guidance

后面继续扩页面时，建议保持这些规则别跑偏。

  • 列表页固定采用：标题区 → toolbar → table/card 的顺序。
  • 详情页固定采用：摘要区 → tabs → 主表单区 + 辅助信息区。
  • 所有模块导航沿用同一 sidebar 样式，不为单页单独造菜单。
  • 所有新页面优先复用 .content-card、.section-header、.section-body、.meta-badge、.empty-state。
  • 如果你后面要继续扩成整套 ERP，我建议下一步补：dashboard、form wizard、modal、offcanvas filter、kanban/list 双视图。
